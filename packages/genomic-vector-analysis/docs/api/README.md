## Contributing

To improve the documentation:

1. Add/update JSDoc comments in source files
2. Follow JSDoc best practices:
   - Include `@example` blocks
   - Document all parameters with `@param`
   - Document return values with `@returns`
   - Add `@remarks` for complexity/performance notes
   - Use `@see` for cross-references
3. Regenerate documentation: `npm run docs`
4. Submit pull request
